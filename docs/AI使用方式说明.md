# AI 使用方式说明

## 一、概述

《命运裂隙》使用 DeepSeek V4 Flash 作为 AI 引擎，在游戏运行时通过 API 实时调用生成游戏内容。AI 不是开发阶段的辅助工具，而是游戏运行时的核心驱动——觉醒体、事件、技能、敌人、里程碑全部由 AI 在游戏过程中即兴创造。

## 二、五大 AI 调用场景

### 1. 觉醒体生成

- **触发时机**：玩家点击"觉醒"按钮时，每局1次
- **超时设置**：8秒
- **输入上下文**：随机选定的觉醒方向（8大方向之一）、难度模式、平衡约束规则
- **输出内容**：觉醒体名称、称号、emoji、背景描述、被动类型（双刃/纯正面/无被动）、被动名称及机制、初始属性（HP/ATK/DEF/EN/CRIT%/CRIT DMG/DODGE%/LUCK）、技能亲和方向、叙事风格标签
- **降级方案**：API 不可用时从该方向的预置觉醒池（每方向3-5个手工审核模板）中随机选取

### 2. 事件生成

- **触发时机**：每触发一个新事件时调用，每层5次
- **超时设置**：15秒
- **输入上下文**：当前层数、本层第几个事件（含节奏提示）、难度模式、玩家觉醒体信息（名称+方向+被动+叙事风格）、玩家等级与完整属性、当前技能列表、已击杀怪物统计、已遭遇主题列表（避免重复）、是否Boss层、是否里程碑事件、幸运值高低提示
- **输出内容**：事件主题、标题、叙事描述、三个选择分支及各自的奖惩结果
- **降级方案**：API 不可用时回退预置事件池（40+事件），界面显示"预设"标记

### 3. 技能生成

- **触发时机**：击杀怪物后判定掉落技能时调用
- **超时设置**：10秒
- **输入上下文**：玩家觉醒体信息（名称+方向+被动）、技能分类（伤害/生存/辅助/能量，由系统根据觉醒亲和加权随机选定）、怪物等级及强度系数、玩家已有技能名称列表（禁止重复）
- **输出内容**：技能名称、分类、emoji、描述、冷却回合数、能量消耗、效果数值、寿命（回合寿命15-40回合 / 层数寿命3-8层）
- **降级方案**：API 不可用时从预置技能模板中随机生成

### 4. 敌人生成

- **触发时机**：战斗事件触发时
- **超时设置**：15秒
- **输入上下文**：当前层数、怪物等级（层数+浮动）、敌人类型（普通/精英/Boss）、难度模式
- **输出内容**：敌人名称、描述、属性（HP/ATK/DEF）、特殊行为描述（Boss特殊技能）
- **降级方案**：API 不可用时按公式 `monsterLevel = floor + random(-1, +2)` 自动生成数值

### 5. 里程碑事件生成

- **触发时机**：每5层触发一次（第5/10/15/20...层）
- **超时设置**：20秒
- **输入上下文**：里程碑类型（试炼之门/远古遗迹/觉醒试炼/深渊裂隙/轮回之塔）、玩家觉醒体信息、玩家状态、难度模式
- **输出内容**：里程碑事件叙事、挑战内容、奖励配置（必须包含技能获取环节）
- **降级方案**：API 不可用时使用预置里程碑模板

## 三、Prompt 模板结构

所有 AI 调用使用结构化 Prompt，包含以下上下文模块：

```
【角色设定】你是一个 Roguelike 游戏的XX系统（觉醒系统/地下城主/技能系统...）
【核心上下文】觉醒方向、玩家状态、难度模式等
【平衡约束】数值上下限、平衡规则、禁止事项
【已有内容】已有技能名称列表、已遭遇主题列表（防止重复）
【输出格式】JSON 字段定义及类型约束
```

关键上下文注入：
- **觉醒方向**：决定生成风格的基本倾向（如"影"方向偏黑暗、神秘、致命）
- **玩家状态**：完整属性面板、技能列表、等级、击杀统计
- **难度信息**：影响事件烈度、敌人强度、奖励丰度
- **节奏提示**：前期铺垫/中期探索/后期高潮，AI 自动调整事件烈度
- **幸运值提示**：高幸运偏向积极结果，低幸运增加风险

## 四、JSON 输出格式与校验机制

### 输出格式

AI 必须输出结构化 JSON，系统对每个字段进行类型检查和数值范围校验。以觉醒体生成为例：

```json
{
  "name": "觉醒体名称（3-6字）",
  "title": "称号",
  "direction": "shadow",
  "emoji": "一个emoji",
  "passiveType": "dual_edge|positive|none",
  "passiveName": "被动名称",
  "passiveMechanics": { "positive": {...}, "negative": {...} },
  "initialStats": {
    "hp": 90, "atk": 12, "def": 3, "energy": 50,
    "critRate": 0.20, "critDamage": 2.5, "dodgeRate": 0.03, "luck": 5
  }
}
```

### 校验规则

- **类型校验**：每个字段检查数据类型（string/number/object）
- **范围校验**：数值字段检查上下限（见下方数值安全机制）
- **枚举校验**：direction、passiveType 等字段必须在允许值列表中
- **唯一性校验**：技能名称不得与已有技能重复
- **完整性校验**：必须包含所有 required 字段

## 五、降级策略

当 DeepSeek API 不可用（超时、网络错误、返回异常）时，系统执行以下降级策略：

1. **首次失败**：自动重试一次
2. **重试仍失败**：从对应的预置内容池中随机选取
   - 觉醒体：每方向预置3-5个手工审核模板
   - 事件：预置事件池40+个
   - 技能：预置技能模板库
   - 敌人：按公式自动生成数值
   - 里程碑：预置里程碑模板
3. **界面标识**：降级内容显示"预设"标记，AI 生成内容显示闪烁的"AI"徽章
4. **结算统计**：游戏结束时展示 AI 生成内容占比

## 六、数值安全机制

### 属性上下限约束

| 属性 | 下限 | 上限 | 说明 |
|------|------|------|------|
| HP | 70 | 130 | 初始不能太脆或太肉 |
| ATK | 7 | 15 | 不能太弱或秒杀 |
| DEF | 2 | 8 | 不能无防御或铁壁 |
| Energy | 40 | 65 | 能量不能太多太少 |
| CritRate | 0.03 | 0.25 | 初始暴击率上限 |
| CritDamage | 1.3 | 2.8 | 初始暴击伤害上限 |
| Luck | 3 | 10 | 初始幸运值上限 |

### 事件数值约束

| 参数 | 下限 | 上限 |
|------|------|------|
| 治疗效果 | 0 | 玩家maxHP x 0.4 |
| 事件伤害 | 0 | 玩家maxHP x 难度系数 |
| 敌人HP | 10 | 基准 x 层数 x 难度 x 1.5 |
| 敌人ATK | 3 | 玩家ATK x 2.0 |

### 平衡评分机制

- **双刃被动**：正面效果强度评分必须约等于负面代价评分
- **纯正面被动**：初始属性总和必须低于双刃型15-20%
- **无被动**：初始属性总和必须高于双刃型25-35%
- **禁止**：无负面且正面效果压倒性强的觉醒

### 校验失败处理

数值越界自动裁剪至合法范围。JSON 解析失败时重新请求一次 AI，第二次仍失败则回退预置内容。

## 七、AI 引擎

本项目使用 **DeepSeek V4 Flash** 作为 AI 引擎，通过其 API 实现低延迟、高并发的实时内容生成。API 配置（地址、Key、模型、温度）通过游戏标题页的设置界面进行管理，默认温度为0.85以保证生成多样性。
