## 命运裂隙 — 优化路线图 v2

> 基于代码深度分析与设计讨论，按实施顺序排列。
> 定位变更：不再是纯前端单文件游戏，可引入更多插件、内容系统甚至多平台支持。

---

### 一、战斗系统重构（核心优先级）

#### 1.1 AI被动机制统一接入

**问题诊断**：当前战斗系统对觉醒被动采用硬编码属性名检查（~21个固定key），但AI生成的 `passiveMechanics` 没有任何schema约束。结果是：

- 24个预设觉醒体中有6个存在"死被动"（定义了但战斗中完全不生效）
  - 暗影君王：整个召唤系统（`summonDur`/`summonDmg`/`defPerSummon`）—— 无召唤机制
  - 命运赌徒：随机倍率系统（`allRandomMult`/`allNegMult`）—— 无通用随机倍率器
  - 光辉圣者：条件增伤（`highEnemyDmg`/`lowSelfDmg`）—— 无对应检查
  - 猩红收割者：击杀buff（`killAtkBuff`/`killDur`）—— 无击杀触发器
  - 死神契约：低血暴击（`lowHpCrit`）和高血能耗（`highHpEnRegen`）—— 未接入
  - 星辰织者：能量上限（`energyMax`）—— 未接入
- AI生成的被动几乎100%使用自定义key名 → 全部变成纯描述文字

**重构方案**：

引入一个**被动效果引擎层**，作为AI输出与战斗代码之间的翻译层。

```
设计思路：
  AI生成 passiveMechanics（自由JSON）
        ↓
  passiveEngine.parse(mechanics)  ← 标准化 + 映射到已知效果类型
        ↓
  battle 调用 passiveEngine.trigger(phase, context)
        ↓
  引擎遍历所有已注册效果，按触发阶段执行
```

**效果类型注册表**（passiveEngine 支持的类型）：

| 效果类型 | 触发阶段 | 描述 | 对应战斗hook点 |
|----------|----------|------|----------------|
| `shield` | battle_start | 战斗开始获得护盾 | runBattle初始化 |
| `hp_regen` | turn_start | 每回合回复HP | 回合开始阶段 |
| `lifesteal` | on_attack | 攻击时吸血 | 玩家攻击结算 |
| `dot_apply` | on_attack | 攻击时给敌人施加持续伤害 | 玩家攻击结算 |
| `chain_strike` | on_attack_count | 每N次攻击触发连锁 | 攻击计数器 |
| `crit_boost` | on_crit | 暴击时额外效果 | 暴击结算 |
| `crit_heal` | on_crit | 暴击时回血 | 暴击结算 |
| `dodge_bonus` | always | 额外闪避率 | 闪避判定 |
| `full_evade` | on_enemy_attack | 概率完全免疫敌方攻击 | 敌方攻击阶段 |
| `freeze_chance` | on_attack | 概率冻结敌方一回合 | 玩家攻击结算 |
| `skill_dmg_mult` | on_skill | 技能伤害倍率 | 技能伤害计算 |
| `heal_amplify` | on_heal | 治疗效果增强 | 治疗结算 |
| `immortal` | on_lethal | 致命一击时存活（每战1次）| 死亡判定 |
| `low_hp_atk` | always | 低血量时攻击力提升 | 伤害计算 |
| `early_battle_atk` | turn_check | 前N回合攻击力提升 | 回合检查 |
| `energy_max_bonus` | always | 能量上限加成 | 能量系统 |
| `kill_buff` | on_kill | 击杀后临时buff | battleWin |
| `conditional_dmg` | on_attack | 条件增伤（敌人高血量等）| 伤害计算 |
| `counter_attack` | on_defend | 防御时反击 | 防御阶段 |
| `reflect` | on_hit | 反弹伤害 | 受伤结算 |
| `summon` | battle_start | 召唤物机制（需新增） | 新增召唤系统 |

**AI提示词改造**：将 `passiveMechanics` 的生成引导为标准化数组格式：

```json
{
  "effects": [
    { "type": "lifesteal", "value": 0.15, "condition": null },
    { "type": "hp_regen", "value": 0.03, "condition": "hp_below_50" },
    { "type": "dot_apply", "value": 0.05, "condition": null }
  ],
  "penalties": [
    { "type": "hp_loss", "value": 0.02, "condition": "per_turn" },
    { "type": "self_burn", "value": 0.03, "condition": "per_turn" }
  ]
}
```

**预设觉醒体修复**：将24个预设体的 `mechanics` 对象全部转换为标准效果数组格式，确保暗影君王的召唤、命运赌徒的随机倍率等都能实际生效。

#### 1.2 战斗节奏优化

**倒计时延长**：`waitForAction()` 中的 `cd-fill` 步进间隔调整：

| 战斗类型 | 倒计时 | 步进间隔 | 说明 |
|----------|--------|----------|------|
| 普通战斗 | 15秒 | 每150ms减1% | 给足思考时间 |
| Boss战 | 20秒 | 每200ms减1% | 复杂决策需要更多时间 |
| 突破战 | 25秒 | 每250ms减1% | 最重要决策给最多时间 |

延长倒计时同时让事件预加载有更充裕的时间完成（`preloadNextEvent()` 在战斗期间后台运行）。

**回合间停顿**：每回合结束后增加300ms延迟，让玩家看清上一回合的伤害/效果数字再进入下一回合。

**战斗日志优化**：
- 关键信息高亮：暴击用金色、闪避用蓝色、buff施加用绿色、debuff用红色
- 伤害数字弹出动画：`-15`、`暴击!-28`、`闪避!` 等浮动文字从目标位置向上飘散

#### 1.3 战斗视觉升级

**区域专属背景**（纯CSS渐变，无需外部图片）：

| 区域类型 | 背景 | 粒子效果 |
|----------|------|----------|
| 野外 | `linear-gradient(180deg, #1a3a1a, #0a1a0a)` | 树叶飘落 |
| 城镇 | `linear-gradient(180deg, #2a2a1a, #1a1a0a)` | 暖色光点 |
| 废墟 | `linear-gradient(180deg, #2a1a3a, #1a0a2a)` | 尘埃漂浮 |
| 道路 | `linear-gradient(180deg, #1a2a3a, #0a1a2a)` | 微风线条 |
| 深渊 | `radial-gradient(circle, #3a0a0a, #0a0a0a)` | 暗红脉动 |
| Boss战 | 全屏暗红脉动 | 震屏+闪光 |

**技能释放特效**（CSS动画）：
- 普通攻击：白色刀光横扫（0.3s）
- 暴击：金色爆闪+屏幕震动（0.5s）
- 治疗：绿色粒子上升（0.4s）
- 火焰技能：橙红色火焰扩散（0.5s）
- 冰霜技能：蓝色结晶扩散（0.5s）
- 暗影技能：紫色烟雾缠绕（0.5s）

**HP条重设计**：
- 双方HP条改为大尺寸横向条，带数字跳动动画
- 受到伤害时数字从HP条位置弹出上浮消散

### 二、世界观构建

> 单独文件：`docs/世界观设定.md`

世界观是一切游戏内容的根基。地图、怪物、事件、道具都必须符合世界观设定。

**核心框架**（待详细编写）：

1. **世界起源**：命运裂隙的本质是什么？为什么存在？裂隙中有哪些文明遗迹？
2. **力量体系**：觉醒者的力量来源、八元素的相互关系、突破的本质
3. **势力与阵营**：裂隙世界中存在哪些组织/势力？它们的立场和目标
4. **历史事件**：塑造当前世界格局的重大事件
5. **物种生态**：裂隙中有哪些种族？它们的社会结构和生存方式

世界观文件完成后，将为以下系统提供约束：
- 地图设计的区域命名和背景描述
- 怪物图鉴的物种设定和分布逻辑
- 事件系统的叙事基调
- 道具系统的来源故事

### 三、大地图设计

基于世界观，设计一个有足够纵深和探索感的大地图。

**设计原则**：
- 地图要有"地理逻辑"：从安全区向外扩展，危险递增
- 地点之间有路径连接，不是随意跳转
- 每个地点有独特的生态和故事
- 地图规模足以支撑多次游戏探索

**初步区域划分**（具体名称待世界观确定后调整）：

```
                    [终焉之塔] ← 最终区域
                       |
              [天空圣域区域群] ← Lv.50+
             /    |     |     \
        [深渊区域群]   [山脉区域群] ← Lv.30-50
             \    |     |     /
              [密林区域群] ← Lv.15-30
             /    |     |     \
        [荒原区域群]   [河谷区域群] ← Lv.5-15
             \    |     |     /
              [起始区域群] ← Lv.1-10
```

每个区域群包含3-5个具体地点，总计约25-30个地点。

**地点属性**：
- 等级范围
- 区域类型（野外/城镇/废墟/道路/深渊）
- 绑定怪物列表（从怪物图鉴中选取）
- 特殊事件池
- 背景描述（1-2句）
- 可探索的隐藏内容

**地图生成提示词**（用于AI生图）：
待世界观确定后，生成一版风格统一的地图prompt。

### 四、怪物图鉴系统

地图确定后，为每个地点设计绑定的怪物。不再依赖AI实时生成怪物，改为预制图鉴+地图调用。

**设计思路**：
- 每个区域群有一套怪物生态（普通怪3-5种 + 精英怪1-2种 + Boss 1种）
- 怪物数据完全预制，包括名称、属性公式、技能、掉落
- 保留AI生成的空间：仅用于特殊事件中的"异变怪物"或"裂隙入侵者"

**怪物数据结构**：

```json
{
  "id": "forest_wolf",
  "name": "幽林暗狼",
  "emoji": "🐺",
  "regions": ["幽暗密林", "猎人小径"],
  "levelRange": [8, 15],
  "type": "normal",
  "baseStats": {
    "hp": 45, "atk": 9, "def": 3,
    "critRate": 0.08, "dodgeRate": 0.12
  },
  "scaling": { "hp": 4, "atk": 1.2, "def": 0.5 },
  "skills": ["撕裂之爪", "嚎叫（恐惧1回合）"],
  "drops": [
    { "item": "wolfFang", "chance": 0.25 },
    { "item": "darkPelt", "chance": 0.10 }
  ],
  "lore": "被裂隙能量侵蚀的林狼，双眼泛紫光，擅长偷袭。"
}
```

**Boss设计**：每个区域群1个Boss，拥有独特机制：
- 专属技能池（3-5个技能，按回合规律释放）
- 阶段转换（HP低于50%时进入狂暴）
- 专属掉落（稀有道具/碎片线索）

**怪物图鉴UI**：
- 游戏中可查看已遭遇的怪物信息
- 显示：名称、外观、属性范围、技能、掉落、弱点
- 未遭遇的怪物显示为???（探索驱动）

### 五、Debuff系统与清除机制

#### 5.1 全局debuff追踪

当前debuff仅在单场战斗内有效（局部变量 `pDefBuff`/`paralyzed` 等）。新增**跨战斗持续**的长期debuff层：

```js
G.longDebuffs = [];  // [{type, value, duration, source}]
G.longBuffs = [];    // [{type, value, duration, source}]
```

- `longDebuffs`/`longBuffs` 在战斗初始化时加载，影响属性计算
- UI上在游戏画面头部显示当前长期效果图标（红色=debuff，绿色=buff）

#### 5.2 长期效果列表

| 效果 | 类型 | 来源 | 持续 |
|------|------|------|------|
| 灵魂裂痕 | debuff -50%暴击率 | Boss特殊技能 | 直到净化 |
| 诅咒之血 | debuff 每场战斗开始-15%HP | 诅咒事件 | 直到净化 |
| 力量封印 | debuff 技能CD+1 | 封印事件 | 直到净化 |
| 命运祝福 | buff +10%全属性 | 特殊事件 | 3场战斗 |
| 虚弱之体 | debuff ATK/DEF各-20% | 死亡惩罚 | 3场战斗 |

#### 5.3 净化事件与道具

**新增事件主题 `purify`**：

```
净化之泉：
  选项A：饮下泉水（清除所有debuff，同时失去所有buff）+ 回15%HP
  选项B：浸泡（清除所有debuff，但扣15%金币）
  选项C：装瓶带走（获得道具「净化之水」，战斗中使用清除debuff）
```

**新增道具**：
- `净化之水`：战斗中清除所有debuff
- `命运解药`：清除debuff + 恢复20%HP
- `狂暴药剂（增强版）`：清除所有buff，换攻击力翻倍3回合

### 六、事件系统增强

配合怪物图鉴和世界观，事件系统需要升级：

1. **事件与地点绑定**：不同地点触发不同主题的事件（城镇→商人/任务，废墟→陷阱/宝藏，深渊→诅咒/奇遇）
2. **事件引用怪物图鉴**：战斗类事件从当前地点的怪物列表中选取敌人，而非AI即兴创造
3. **事件链/支线任务**：部分事件形成2-3步的小故事链，跨月份推进
4. **debuff相关事件**：净化事件、诅咒事件、封印事件加入事件池

### 七、实施优先级

| 顺序 | 模块 | 工作量 | 核心改动 |
|------|------|--------|----------|
| 1 | 被动效果引擎 + 预设修复 | 8h | passiveEngine层 + 24预设转换 + AI提示词改造 |
| 2 | 战斗节奏 + 视觉升级 | 5h | 倒计时分级 + 回合停顿 + CSS特效 + 伤害弹出 |
| 3 | 世界观设定 | 3h | 独立文档，约束后续所有内容 |
| 4 | 大地图设计 | 4h | 25-30地点 + 路径连接 + 区域属性 |
| 5 | 怪物图鉴 | 6h | ~60种怪物 + Boss + 数据结构 + 图鉴UI |
| 6 | Debuff系统 + 净化 | 3h | 全局debuff层 + 净化事件/道具 |
| 7 | 事件系统增强 | 4h | 地点绑定 + 图鉴引用 + 事件链 |

**建议**：1-2可并行（被动引擎和视觉互不依赖），3必须先于4和5（世界观约束地图和怪物），6-7在4-5之后。
